# Offline Password Cracking – PortSwigger Authentication Lab

## Lab Overview
Completed a PortSwigger Web Security Academy lab focused on Offline Password Cracking and Authentication Security. The lab demonstrated how exposed authentication-related data and weak credential handling can lead to account compromise.

---

## What I Did
- Intercepted authentication traffic using Burp Suite
- Analyzed login requests and responses
- Identified sensitive authentication data exposure
- Extracted information useful for offline password analysis
- Performed password testing using a wordlist approach
- Reused recovered credentials to successfully log in
- Completed the lab by exploiting the authentication weakness

---

## Concepts Used
- Authentication Security Testing
- Offline Password Cracking
- Credential Exposure Analysis
- Login Workflow Analysis
- HTTP Request & Response Inspection
- Web Application Security Testing
- Password Security Fundamentals

---

## Tools Used
- Burp Suite
- PortSwigger Web Security Academy
- Browser Developer Tools
- Wordlist-based Password Testing

---

## What I Learned
- Difference between online and offline password attacks
- Importance of secure password hashing and salting
- How weak authentication implementations increase risk
- Practical use of Burp Suite for authentication testing
- Better understanding of login flows and attack surfaces
- Importance of strong password policies and MFA

---

## Key Takeaway
This lab showed how attackers can compromise accounts without directly brute-forcing login pages when sensitive authentication data becomes exposed. Strong hashing algorithms, MFA, rate limiting, and secure authentication design are essential defenses.
